**Статус:** ЧЕРНОВИК
**Архитектор:** Antigravity (Google DeepMind)
**Дата:** 11.01.2026

---

## 1. РЕЗЮМЕ
Цель состоит в том, чтобы трансформировать Aura Mebel из "Чат-бота с инструментами" в **Генеративную Агентную ОС**.
Система будет не просто *отвечать* на вопросы; она будет *конструировать* интерфейсы, *предугадывать* потребности и автономно *оркестрировать* сложные многоэтапные рабочие процессы (Поиск -> Визуализация -> Проверка -> Покупка).

### Ключевые Инновации
1.  **Генеративный UI (GenUI):** Агент не выводит текст; он транслирует React-компоненты (`<Carousel />`, `<ARPreview />`, `<PriceCard />`) непосредственно в поток чата.
2.  **Нейро-Символическое Ядро:** Объединение интуиции LLM (Креативность) с детерминированной логикой (Математика, Фильтрация, Ценообразование).
3.  **Проактивные "Теневые" Агенты:** Агенты, которые наблюдают и действуют без прямых подсказок (например, "Контроль Качества", проверяющий каждый сгенерированный ответ).

## 2. АРХИТЕКТУРА "УЛЕЙ" (HIVE)

Мы переходим от линейного "Оркестратора" к **Архитектуре Графа Состояний** (вдохновленной LangGraph/StateMachine).

### 2.1 Координатор (Узел-Супервизор)
*   **Роль:** Диспетчер. Хранит "Глобальное Состояние" разговора.
*   **Инновация:** Использует **Скрытый Поток Мыслей** для рассуждения перед маршрутизацией.
*   **Логика Маршрутизации:**
    *   *needs_visuals* -> **Визуальный Дизайнер**
    *   *needs_products* -> **Поисковик Каталога**
    *   *needs_math* -> **Калькулятор Пространства**
    *   *needs_safety* -> **Хранитель Бренда**

### 2.2 Специализированные Агенты (Рабочие)

#### 🎨 A. Визуальный Дизайнер (Vision-Language Model)
*   **Технология:** Gemini Flash 2.0 / GPT-4o
*   **Вход:** Фото от пользователя / Поток с камеры.
*   **Выход:** `RoomContext` (Освещение, Стиль, Цветовая палитра).
*   **Инновация:** Может "рисовать" ограничивающие рамки (bounding boxes) на загруженном изображении, показывая, где должна стоять мебель.

#### 🛍️ B. Поисковик Каталога (Vector + SQL)
*   **Технология:** Эмбеддинг-модель (Text-to-Vector) + PostgreSQL (Метаданные).
*   **Возможность:** "Семантическое Хеширование".
    *   Пользователь: "Хочу кресло, которое обнимает."
    *   SQL: `category='chair'`
    *   Vector: `search(query='cozy, enveloping, soft', threshold=0.8)`
*   **Выход:** Возвращает сырой JSON Координатору, который рендерит `<ProductGrid />`.

#### 📐 C. Калькулятор Пространства (Символическая Логика)
*   **Технология:** Чистый TypeScript (Без LLM).
*   **Зачем?** LLM плохо считают. Для этого мы используем код.
*   **Роль:** Проверяет размеры. "Влезет ли диван 220см в простенок 200см?" -> **Детерминированное НЕТ**.

#### 🛡️ D. Хранитель Бренда (Теневой Агент)
*   **Роль:** Цензор.
*   **Поведение:** Работает параллельно с каждой генерацией LLM.
*   **Чек-лист:**
    1.  Обнаружение устаревшего бренда ("Labelcom").
    2.  Проверка на галлюцинации (Придумал ли LLM скидку?).
    3.  Полировка тона (Теплый Минимализм).

## 3. СТРАТЕГИЯ РЕАЛИЗАЦИИ: "React Server Components (RSC) + AI Streams"

Мы будем использовать подход **Vercel AI SDK 3.0** для стриминга UI.

### 3.1 Концепция "Потока" (Stream)
Вместо ожидания полного ответа, Агент действует так:

`Пользователь: "Покажи современные диваны"`

**T=0ms**: `<Spinner label="Координатор думает..." />`
**T=200ms**: `Координатор`: "Ищу современные диваны в нашем каталоге."
**T=300ms**: `<SkeletonGrid />` (Стриминг вызова инструмента...)
**T=800ms**: `<ProductCarousel items={[...]} />` (Отрисованный компонент)
**T=900ms**: `Координатор`: "Нашел. Нажмите 'Посмотреть в комнате', чтобы увидеть их в AR."

### 3.2 Структура Директорий (Папка "Мозг")

```text
lib/
└── ai/
    ├── actions.tsx         // Server Actions (Точка входа)
    ├── registry.tsx        // Реестр UI компонентов (GenUI)
    ├── state.ts            // Управление глобальным состоянием
    └── agents/
        ├── supervisor.ts   // Маршрутизатор
        ├── vision.ts       // Анализ изображений
        ├── catalog.ts      // RAG / Поиск
        └── guardian.ts     // Безопасность/Бренд
```

## 5. ПРИМЕР "МЫСЛИТЕЛЬНОГО ПРОЦЕССА" (Логи)

> **Пользователь:** "Мне нужен стол для этого угла." (Загружает фото)
>
> 1.  **Супервизор**: Получает изображение. Определяет интент `visual_consultation`.
> 2.  **Маршрутизатор**: Передача -> `VisionAgent`.
> 3.  **VisionAgent**: 
>     *   *Анализ*: "Стандартное освещение, доступная ширина ок. 1.5м. Стиль: Индустриальный."
>     *   *Возврат*: `query_params: { style: 'industrial', max_width: 150 }`
> 4.  **Супервизор**: Передача -> `CatalogAgent`.
> 5.  **CatalogAgent**:
>     *   *Поиск*: Найдено 3 стола.
>     *   *SpaceCalc*: Стол A (140см) - ОК. Стол B (160см) - ОТКЛОНЕН.
> 6.  **Супервизор**: streamUI(`<ProductCarousel products={[Стол A]} />`).
